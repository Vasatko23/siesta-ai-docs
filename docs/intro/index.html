Introduction

The product specification provides a comprehensive description of the SIESTA AI platform, a tool designed for companies that want to fully leverage the potential of artificial intelligence in their daily operations. The platform builds on the capabilities of current large language models but brings a number of key features essential for professional and secure deployment in enterprise environments.

SIESTA AI enables companies to connect their own knowledge base with powerful AI models in a way that is secure, scalable, and easily integrated into business processes. Users interact with the platform, which draws answers from connected knowledge sources. The platform is model-agnostic and supports both cloud and self-hosted deployments, giving companies full control over where and how their data flows.

The main areas the platform focuses on are:

Assistants​

Assistants are the fundamental building blocks of the Siesta AI platform. Each assistant represents a separate digital agent based on a language model, connected to specific data, equipped with its own instructions, and configured to serve its specific purpose. The platform allows the creation of countless such agents and their centralized management. Each assistant can be designated for a different team, department, language, or business scenario.

Users can create assistants from scratch or from prepared templates, set their behavior, connect datasets, define permissions, and adjust their outputs. The configuration of each assistant includes its own system instructions, model selection, output format, creativity level, data access, and options for public or internal deployment.

Each assistant has its own detail page, where its activity, performance, feedback, and change history can be monitored. This detail serves as a central place for its management and is divided into several functional tabs covering all areas, from basic information through analytics and feedback to change history.

Assistants can be accessed via chat, deployed as a public widget, integrated into Slack, Teams, or other communication channels, or used directly via API. The platform also allows connecting assistants to specific tools they can actively use.

Integrations, Tools, and Automation​

One of the key areas of the Siesta AI platform is the ability to connect assistants to external tools, systems, and services. Thanks to this feature, assistants are not just conversational interfaces but active elements of digital infrastructure that can perform specific tasks, transfer data, or respond to real-time events.

Each assistant can be assigned one or more tools to use during interaction or automatically in the background. For example, integration with a calendar, file storage, internal system via API, or external webhook. Tools are managed centrally within the Tools interface and can be assigned across assistants.

Data​

One of the main benefits of the Siesta AI platform is the ability to connect artificial intelligence to the specific knowledge of an organization. Assistants do not respond based on a general model but draw information from precisely defined datasets. These datasets can include both uploaded documents and connections to live systems and knowledge bases used by the company in daily operations.

The platform allows the creation, management, and configuration of so-called datasets—logically separated sets of information that are then assigned to specific assistants.

Each assistant can be assigned which datasets it is allowed to use, thus precisely controlling what information it has access to. This approach not only increases the relevance of responses but also ensures a high level of security, as no assistant has access to data that has not been explicitly assigned to it.

Smart Chat​

The Siesta AI platform provides a unified interface for communication with AI assistants, available to both internal users and external visitors. The goal of this part of the system is to create an environment where users can naturally communicate with individual assistants, receive relevant answers based on company data, and provide feedback to further improve their behavior.

The platform also includes so-called smart chat, a public anonymous interface that can be embedded on websites or internal company portals. This chat is available without login and can be used, for example, for customer support, career communication, or as a tool for answering frequently asked questions. The public chat plugin is fully integrated with the platform, maintains all security principles, enables feedback collection, and is connected to audit logs.

The platform also supports access to assistants via API or integration into commonly used tools such as Slack or Microsoft Teams. This allows users to communicate with assistants directly from the environment they use daily, without the need to log into a new application. Each assistant can be deployed in multiple channels simultaneously, while logic, data, and behavior remain consistent.

User and Access Management​

The Siesta AI platform enables detailed management of users, their roles, and access permissions. Each user in the system is assigned a role that determines which functions, data, and assistants they can access.

Administrators can create and edit user accounts, set permission granularity, and precisely define who can edit assistants, work with datasets, view feedback, or access system settings.

Access can be controlled not only at the platform level but also within individual components, such as at the level of a specific assistant or dataset.

Feedback Loop​

One of the key features of the Siesta AI platform is the ability to work with user feedback and continuously improve the behavior of individual assistants based on it. Each assistant's response can be rated by the user—either positively or negatively. In the case of negative feedback, the user can attach a comment explaining why the answer was unclear, incorrect, or unhelpful.

Security and Audit Logs​

Every important operation in the system—whether editing an assistant, changing permissions, working with data, or generating responses—is recorded and traceable.

The system includes detailed audit logs that capture who did what, when, and on what. These records are available to administrators in a clear format and allow retrospective control over all changes in the system.

The platform also enables API key management, access control via roles, and support for single sign-on (SSO). The security policy meets the needs of organizations that emphasize data control, auditability, and operational reliability.

Analytics and Reporting​

The Siesta AI platform includes built-in tools for monitoring traffic, user behavior, and the performance of individual assistants. The goal of this part of the system is to provide administrators and managers with an overview of how assistants are used, what impact they have, and where there is room for improvement.
